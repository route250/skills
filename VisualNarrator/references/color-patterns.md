# 色彩パターン（用途別・視覚効果別）

この資料は、目的に応じた配色を選ぶための実務基準。

## 1. データ表現の基本パターン
| 用途 | 推奨パレット | 視覚効果 | 注意点 |
|---|---|---|---|
| 量の増減（低→高） | Sequential（連続） | 強弱・密度が直感的 | 明度差を主軸にする |
| 正負・基準差（±） | Diverging（発散） | 中央基準からの逸脱が分かる | 中央値の意味を明記 |
| カテゴリ比較 | Qualitative（定性的） | 分類の識別性が高い | 色数を必要最小限にする |

## 1.1 コンセプトカラーの決め方
1. 作品の主目的を1語で定義する（例: 信頼、警戒、成長）。
2. 目的に対応するコンセプトカラーを1色だけ選ぶ。
3. その色から `primary` / `primary-weak` / `primary-strong` を派生させる。
4. 背景・図形塗り・枠線・文字のトークンに割り当てる。
5. 全シーンで同じ意味の要素には同じトークンを使う。
- コンセプトカラー未定義で制作を始めると、ほぼ確実に単色の雑な動画になる。

## 2. シーン目的別の推奨
| シーン目的 | 色戦略 | 期待する効果 |
|---|---|---|
| 導入（全体像） | 低彩度 + 1アクセント | 認知負荷を抑え、焦点を作る |
| 警告・リスク提示 | 暖色アクセント（赤/橙）を限定使用 | 注意喚起を強化 |
| 比較説明 | ニュートラル地 + 対比2色 | 差分の可視化 |
| 手順説明 | 同系色の明度段階 | 連続性・順序感 |
| 締め・結論 | 主ブランド色 + 高コントラスト文字 | 記憶定着 |

## 3. 可読性とアクセシビリティ
- 通常文字のコントラスト比は少なくとも 4.5:1。
- 大きい文字（約24px以上相当）は少なくとも 3:1。
- 色だけで意味を伝えない。形・位置・ラベルを併用する。
- 1080p基準の目安: 本文 38px 以上、注釈 28px 以上、見出し 52px 以上。
- 細線フォントの多用を避ける。日本語本文は `fontWeight` 500-700 を優先する。

## 4. 実装メモ
- 先にグレースケールで階層を作り、最後にアクセント色を足す。
- 同一シーンで意味の異なる赤を複数使わない。
- 色覚多様性に備え、赤緑依存の符号化を避ける。
- 背景色、図形塗り、枠線色、文字色をトークン化して `design/visual-spec.md` に固定する。
- 枠線は背景との明度差を確保する。背景と同化する枠線は装飾コストだけ増やす。

## 4.1 カラートークン例（最小構成）
- `bg/base`: 全体背景
- `bg/panel`: カード・図形背景
- `stroke/default`: 枠線
- `text/primary`: 主本文
- `text/secondary`: 補助本文
- `accent/primary`: 重要値・注目点
- `status/risk`: 警告・リスク表示
