# インフォグラフィック典型パターン

この資料は、メッセージ種別ごとに最適な図版型と視覚効果を選ぶための早見表。

## 1. 型の選定
| メッセージ | 推奨型 | 典型構成 | 視覚効果 | 避けるべき失敗 |
|---|---|---|---|---|
| 比較（A vs B） | バー比較 / カード比較 | 左右2〜4ブロック | 差分が即時に知覚される | 円グラフ多用で差が読めない |
| 時系列の変化 | タイムライン / 折れ線 | 左→右で時間進行 | 変化の方向と速度が伝わる | 目盛り不統一で誤読 |
| 構成比 | 積み上げバー / ドーナツ（少数カテゴリのみ） | 合計100%明示 | 全体内の比率を把握しやすい | カテゴリ過多で判別不能 |
| 因果関係 | フロー図 / 矢印ネットワーク | 原因→結果を段階分割 | ロジックの連鎖が見える | 線が交差して経路不明 |
| 階層構造 | ツリー / ネストボックス | 上位→下位の段差表現 | 支配関係が明確 | 階層と順序を混同 |
| プロセス説明 | ステップ図（1-2-3） | 各段を同一フォーマット | 手順の再現性が上がる | 1画面に分岐を詰め込む |

### 1.1 型選定時のRemotion実装ガイド
| 型 | 実装の主軸 | 実装メモ |
|---|---|---|
| バー比較 / カード比較 | `AbsoluteFill` + グリッド配置 + `Sequence` スタガー | 差分を強調する要素だけ遅延表示する。 |
| タイムライン / 折れ線 | 左右固定レイアウト + `interpolate()` で値遷移 | 軸と目盛りは先に固定表示し、線だけアニメーションする。 |
| 積み上げバー / ドーナツ | セグメントごとの色トークン固定 + `spring()`/`interpolate()` | 項目順をシーン間で変えない。比較不能になる。 |
| フロー図 / 矢印ネットワーク | ノード先置き + 矢印を段階表示 | 線の交差を回避するためノード座標を先に確定する。 |
| ツリー / ネストボックス | 親から子へ `Sequence` 展開 | 同時表示しすぎると階層が崩れる。段階展開を優先する。 |
| ステップ図 | 1ステップ1 `Sequence` + 進行インジケータ固定 | 進捗表示を固定し、本文だけ差し替える。 |

## 2. 画面サイズと占有率ガイド（用途別）
| 用途 | 想定サイズ | メイン図版幅 | 主要ラベル高さ | 余白 | 最小文字サイズ目安 |
|---|---|---|---|---|---|
| プレゼン動画（標準） | 16:9（1920x1080） | 60-80% | 12-18% | 5-8% | 本文 38px / 補助 28px |
| 研修・投影（遠距離視認） | 16:9（1920x1080） | 55-75% | 14-22% | 6-10% | 本文 44px / 補助 32px |
| 縦動画（SNS） | 9:16（1080x1920） | 78-92% | 10-16% | 4-6% | 本文 48px / 補助 36px |
- 1シーン内の主要オブジェクト数は 3-7 個に制限する。
- 表紙・オープニング・クロージングは本文より情報密度を下げ、識別可能な余白を確保する。

### 2.1 画像有無でのレイアウト選択
| 条件 | 推奨レイアウト | 画面配分の目安 | 向いている場面 |
|---|---|---|---|
| 画像なし | 全面レイアウト（図版を画面全体で使用） | 図版 85-100% | 数値比較、手順説明、結論強調 |
| 画像あり | 右画像・左情報 | 左 45-60% / 右 40-55% | 左から読む文脈で説明主体 |
| 画像あり | 左画像・右情報 | 左 40-55% / 右 45-60% | 右に数値・結論を置く構成 |
| 画像あり | 上画像・下情報 | 上 45-65% / 下 35-55% | 写真の文脈説明、図解補足 |
| 画像あり | 下画像・上情報 | 上 35-55% / 下 45-65% | 見出し先行、画像で着地 |
| 画像あり | 中央ヒーロー画像 + 周辺注釈 | 中央 50-75% | キービジュアル訴求、導入/締め |
- 固定で「左60%」を既定にしない。目的と素材で毎シーン選ぶ。
- 画像がないのに左右2カラムにするのは、余白の浪費になりやすい。

### 2.2 配置決定ルール
1. 画像の情報価値が高い場合は、画像面積を先に確保する。
2. 数値比較が主目的なら、画像を外して全面図版を優先する。
3. 表紙・オープニング・クロージングは中央ヒーローまたは全面型を優先する。
4. 同じ動画内で配置パターンを毎シーン変えすぎない（2-3種まで）。

## 3. セーフゾーン
- タイトル/重要文字はタイトルセーフ領域（内側80%）に収める。
- 動きのある要素はアクションセーフ領域（内側90%）を目安に配置する。

## 4. 運用ルール
- 1シーン1メッセージを厳守する。
- 文字を増やして説明しない。図版の比較軸を増やして説明する。
- 同一動画内で図版文法を頻繁に変えない。
- 表紙・オープニング・本文・クロージングでレイアウト系統を統一し、突然別UIにしない。
